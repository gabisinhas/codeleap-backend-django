# Organização MVC

- models/: modelos de dados (ex: post.py)
- views/: views e endpoints da API
- controllers/: lógica de controle (opcional, para lógica fora das views)
- serializers/: serializers DRF
- templates/: templates HTML (se usar)
- static/: arquivos estáticos
- tests/: testes automatizados
- docs/: documentação do app

Adapte os arquivos para dentro dessas pastas conforme o crescimento do projeto.
